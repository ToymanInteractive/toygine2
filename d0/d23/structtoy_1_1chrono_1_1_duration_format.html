Detailed Description

template<typename Rep, typename Period>
class toy::chrono::DurationFormat< Rep, Period >

Pairs a toy::chrono::Duration with a null-terminated display pattern.

Passed to toy::OStringStream::operator<< to format the duration into a human-readable clock representation. The pattern is scanned character-by-character; recognised tokens are replaced with the corresponding time component derived from a millisecond decomposition; all other characters are emitted verbatim.

Template Parameters
RepArithmetic representation type of the duration tick count.
Periodstd::ratio specifying the tick period relative to one second.

Key Features

  • Flexible patterns: any combination of h, m, s, z tokens with arbitrary literal separators.
  • Millisecond precision: decomposes duration down to milliseconds; sub-millisecond ticks are truncated.
  • Negative durations: prefixes the formatted output with '-'; absolute value is formatted normally.
  • Literal passthrough: non-token characters (':', '.', spaces, labels) are emitted unchanged.

Pattern Tokens

Token Component Behaviour
h hours no leading zero (e.g. 9)
hh hours always 2 digits (e.g. 09)
m minutes no leading zero (e.g. 3)
mm minutes always 2 digits (e.g. 03)
s seconds no leading zero (e.g. 5)
ss seconds always 2 digits (e.g. 05)
z milliseconds no leading zero (e.g. 42)
zzz milliseconds always 3 digits (e.g. 042)

Usage Example

#include "core.hpp"
toy::chrono::ClockSource clock;
toy::chrono::Stopwatch sw;
// ... work ...
const auto fmt = toy::chrono::DurationFormat{"hh:mm:ss.zzz", sw.elapsed()};
toy::OStringStream<toy::FixedString<32>> stream;
stream << fmt; // e.g. "00:00:01.042"

Performance Characteristics

  • Construction: O(1); no computation at construction time.
  • Formatting: O(n) where n is the length of pattern; one duration_cast to milliseconds then a single pass through the pattern string.
  • Memory usage: 8–16 bytes depending on Rep and pointer size; no heap allocation.

Safety Guarantees

  • Pattern lifetime: pattern must remain valid for the duration of the format operation; string literals and static-storage pointers satisfy this. Dangling pointers cause undefined behaviour.
  • Exception safety: all operations are noexcept.
