Detailed Description

Pairs a toy::chrono::CalendarTime with a null-terminated display pattern.

Passed to toy::OStringStream::operator<< to format the calendar date and time into a human-readable representation. The pattern is scanned character-by-character; recognised tokens are replaced with the corresponding field of the toy::chrono::CalendarTime; all other characters are emitted verbatim.

Key Features

  • Flexible patterns: any combination of y, M, d, h, m, s, z tokens with arbitrary literal separators.
  • Literal passthrough: non-token characters ('-', ':', '.', 'T', spaces, labels) are emitted unchanged.

Pattern Tokens

Token Component Behaviour
y year no leading zero (e.g. 2026)
yy year last 2 digits, zero-padded (e.g. 26)
yyyy year always 4 digits, zero-padded (e.g. 2026)
M month no leading zero (e.g. 5)
MM month always 2 digits (e.g. 05)
d day no leading zero (e.g. 7)
dd day always 2 digits (e.g. 07)
h hour no leading zero (e.g. 9)
hh hour always 2 digits (e.g. 09)
m minute no leading zero (e.g. 3)
mm minute always 2 digits (e.g. 03)
s second no leading zero (e.g. 5)
ss second always 2 digits (e.g. 05)
z millisecond no leading zero (e.g. 42)
zzz millisecond always 3 digits (e.g. 042)

Usage Example

#include "core.hpp"
const toy::chrono::CalendarTime ct = toy::chrono::SystemClock::now();
toy::OStringStream<toy::FixedString<32>> stream;
stream << toy::chrono::CalendarTimeFormat{"yyyy-MM-dd hh:mm:ss.zzz", ct}; // e.g. "2026-05-20 14:30:45.123"

Performance Characteristics

  • Construction: O(1); no computation at construction time.
  • Formatting: O(n) where n is the length of pattern; one pass through the pattern string.
  • Memory usage: 12 bytes (toy::CStringView pointer/length plus 10-byte toy::chrono::CalendarTime); no heap allocation.

Safety Guarantees

  • Pattern lifetime: pattern must remain valid for the duration of the format operation; string literals and static-storage pointers satisfy this. Dangling pointers cause undefined behaviour.
  • Exception safety: all operations are noexcept.
Note
Month token uses uppercase M to disambiguate from minutes; all other tokens are lowercase as in toy::chrono::DurationFormat.
yy emits the last two digits of the year (e.g. 2026 → 26).
