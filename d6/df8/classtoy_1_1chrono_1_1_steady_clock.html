Detailed Description

Stateless monotonic clock that satisfies the TrivialClock concept.

A stateless facade over the active toy::chrono::ClockSource. All methods delegate to the source registered by the most recently constructed, still-living toy::chrono::ClockSource. The tick period is fixed at compile time via toy::chrono::c_steadyClockPeriodDenominator.

Key Features

  • TrivialClock-compatible: provides rep, period, duration, time_point, is_steady, and now() as required by the std::chrono::TrivialClock named requirements.
  • Stateless: no instance data; all methods are static. Suitable for use as a clock type parameter in std::chrono templates.
  • Compile-time period: period is derived from toy::chrono::c_steadyClockPeriodDenominator and resolved at compile time.
  • Thin wrapper: delegates directly to the active source; no additional state and no exceptions.

Usage Example

#include "core.hpp"
int main() {
toy::chrono::ClockSource clock;
const auto t0 = toy::chrono::SteadyClock::now();
// ... work ...
const auto t1 = toy::chrono::SteadyClock::now();
const auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>(t1 - t0);
}

Performance Characteristics

  • now(): O(1); one counter read plus a constant unit conversion when the source frequency differs from period::den.
  • nowTicks(): O(1); one counter read with no unit conversion.
  • frequency(): O(1); one field load from the active toy::chrono::ClockSource.
  • Memory usage: 0 bytes (stateless type).

Safety Guarantees

  • Precondition: a toy::chrono::ClockSource must be alive for the duration of any call; behaviour is undefined if no source is active.
  • Monotonicity: now() is monotonically non-decreasing on a single thread; not guaranteed across threads without synchronisation.
  • Exception safety: all operations are noexcept.
Note
is_steady is true: the underlying counter is never adjusted after the source is constructed.
