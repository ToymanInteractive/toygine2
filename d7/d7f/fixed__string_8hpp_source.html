30/*!
31 \class FixedString
32 \brief Template string class with fixed-size character buffer.
33
34 FixedString is a lightweight, high-performance string class that uses a fixed-size character buffer allocated on the
35 stack. It provides a std::string-like interface while avoiding dynamic memory allocation, making it suitable for
36 embedded systems, real-time applications, and performance-critical code where memory allocation overhead must be
37 minimized.
38
39 \tparam allocatedSize The size of the internal character buffer, including space for the null terminator.
40 Must be greater than zero.
41
42 \section features Key Features
43
44 - ⚙️ **Zero Dynamic Allocation**: All memory is allocated on the stack at compile time
45 - 🔧 **constexpr Support**: Most operations can be evaluated at compile time
46 - 🛡️ **Exception Safety**: All operations are noexcept
47 - 🔗 **STL Compatibility**: Provides std::string-like interface
48 - 🌍 **UTF-8 Support**: Built-in UTF-8 character counting and manipulation
49 - 🧬 **Template Metaprogramming**: Uses C++20 concepts for type safety
50
51 \section usage Usage Example
52
53 \code
54 #include "fixed_string.hpp"
55
56 // Create a string with 32-character capacity
57 toy::FixedString<32> str("Hello, World!");
58
59 // Append more content
60 str += " This is a test.";
61
62 // Use in constexpr context
63 constexpr auto greeting = toy::FixedString<16>("Hello");
64 constexpr auto world = toy::FixedString<16>("World");
65 constexpr auto combined = greeting + " " + world;
66 \endcode
67
68 \section performance Performance Characteristics
69
70 - ⚙️ **Construction**: O(n) where n is the length of the source string
71 - 📝 **Assignment**: O(n) where n is the length of the source string
72 - 🔗 **Concatenation**: O(n) where n is the length of the appended string
73 - 🔍 **Search Operations**: O(n*m) where n is the string length and m is the pattern length
74 - 💾 **Memory Usage**: Fixed at compile time, no heap allocation
75 - ⚡ **Cache Performance**: Excellent due to stack allocation and contiguous memory layout
76 - 📋 **Copy Performance**: Fast due to memcpy/memmove optimizations
77 - 🎯 **String Operations**: Optimized for common cases (single character, empty strings)
78
79 \section safety Safety Guarantees
80
81 - 🛡️ **Contracts & Debug Checks**: Bounds/capacity are asserted in debug; in production, violating preconditions is UB
82 - 🔒 **Null Pointer Safety**: All C-string operations validate pointers via assertions in debug
83 - 📐 **Type Safety**: Template parameters and concepts prevent invalid usage
84 - ⚠️ **Exception Safety**: All operations are noexcept, no exceptions thrown
85
86 \section compatibility Compatibility
87
88 - 🆕 **C++23**: Requires C++23 or later for full functionality
89 - 🔗 **STL Integration**: Compatible with STL algorithms and containers
90 - 🌐 **Cross-Platform**: Works on all platforms supported by the compiler
91 - 🔧 **Embedded Systems**: Suitable for resource-constrained environments
92
93 \note The internal buffer size is allocatedSize, but the maximum string length is allocatedSize - 1 to account for the
94 null terminator.
95
96 \see std::string
97 \see StringLike
98*/
