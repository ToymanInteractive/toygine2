30/*!
31 \struct Version
32 \brief Application version information structure.
33
34 Version represents a semantic version number following the major.minor.maintenance.revision format. This structure is
35 designed for compile-time version management and provides a simple way to track application versions throughout the
36 development lifecycle.
37
38 \section features Key Features
39
40 - ⚙️ **Semantic Versioning**: Follows major.minor.maintenance.revision format
41 - 🔧 **constexpr Support**: All operations can be evaluated at compile time
42 - 🛡️ **Exception Safety**: All operations are noexcept
43 - 🔗 **Simple Structure**: Plain struct with public members for easy access
44 - 🎯 **Version Management**: Designed for application version tracking
45 - 🧬 **Type Safety**: Uses std::uint32_t for all version components
46 - 🌍 **Standard Compliance**: Follows semantic versioning standards
47 - 📏 **Immutable Design**: Structure designed for compile-time version constants
48
49 \section usage Usage Example
50
51 \code
52 #include "app.hpp"
53
54 // Default version (0.0.0.0)
55 constexpr toy::app::Version version;
56
57 // Custom version
58 constexpr toy::app::Version customVersion{1, 2, 3, 4};
59
60 // Access version components
61 std::cout << "Version: " << customVersion.major << "."
62 << customVersion.minor << "."
63 << customVersion.maintenance << "."
64 << customVersion.revision << std::endl;
65 \endcode
66
67 \section performance Performance Characteristics
68
69 - ⚙️ **Construction**: O(1) constant time
70 - 📝 **Access**: O(1) constant time for all members
71 - 📝 **Assignment**: O(1) constant time for member assignment
72 - 🔗 **Comparison**: O(1) constant time for version comparison
73 - 💾 **Memory Usage**: 16 bytes (4 * std::uint32_t)
74 - ⚡ **Cache Performance**: Excellent due to small size and contiguous layout
75 - 📋 **Copy Performance**: Fast due to simple integer copying
76 - 🎯 **Version Operations**: Optimized for compile-time evaluation
77
78 \section safety Safety Guarantees
79
80 - 🛡️ **Bounds Safety**: All operations are bounds-safe
81 - 📐 **Type Safety**: Strong typing with std::uint32_t components
82 - ⚠️ **Exception Safety**: All operations are noexcept, no exceptions thrown
83 - 🔒 **Memory Safety**: No dynamic allocation, stack-only structure
84
85 \section compatibility Compatibility
86
87 - 🌐 **Cross-Platform**: Works on all platforms supported by the compiler
88 - 🔧 **Embedded Systems**: Suitable for resource-constrained environments
89 - 📱 **Mobile Platforms**: Lightweight and efficient for mobile applications
90
91 \note This structure is designed for compile-time version management. For runtime version parsing from strings,
92 consider implementing additional utility functions.
93
94 \see Semantic Versioning (https://semver.org/)
95*/
96struct Version {
97 /*!
98 \brief Major version number.
99 \details Indicates incompatible API changes.
100 */
101 std::uint32_t major = 0;
102
103 /*!
104 \brief Minor version number.
105 \details Indicates backward-compatible functionality additions.
106 */
107 std::uint32_t minor = 0;
108
109 /*!
110 \brief Maintenance version number.
111 \details Indicates backward-compatible bug fixes.
112 */
113 std::uint32_t maintenance = 0;
114
115 /*!
116 \brief Revision number.
117 \details Indicates build number or patch level.
118 */
119 std::uint32_t revision = 0;
120};

122/*!
123 \brief Equality operator for two versions.
124
125 Compares two versions for exact equality of all components.
126
127 \param lhs The left-hand side version.
128 \param rhs The right-hand side version.
129
130 \return true if all version components are identical, false otherwise.

137/*!
138 \brief Three-way comparison operator for two versions.
139
140 Compares two versions using lexicographical ordering following semantic versioning principles. The comparison

142
143 \param lhs The left-hand side version.
144 \param rhs The right-hand side version.

148*/
