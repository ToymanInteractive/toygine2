Detailed Description

Process-wide singleton transport for log records.

Owns the active toy::log::ISink and the timestamp policy used to stamp each pushed record. Constructed once per process (typically as a member of toy::CoreApplication); a debug assertion fires if a second instance is created while one is already active. LOG_* macros call Backend::instance().push() to format the message on the caller's stack and dispatch the resulting record synchronously to the installed sink.

Key Features

  • RAII singleton: lifetime tied to a stack or member object; debug assertion enforces uniqueness.
  • Pluggable sink: setSink() swaps the destination at runtime; null sink silently drops records.
  • Pluggable timestamp: setTimestampPolicy() installs a callback; defaults to a zero-returning stub.
  • Synchronous dispatch: push() formats and writes inline; no ring buffer or background thread in v1.
  • No allocation: toy::log::Record is constructed on the caller's stack.
  • Noexcept: every method is noexcept; the log path is exception-free.

Usage Example

#include "core.hpp"
toy::log::Backend backend; // RAII; registers as active
toy::ConsoleSink sink;
backend.setSink(&sink);
static constexpr toy::log::Metadata meta{"player at ({}, {})", __FILE__, __LINE__, toy::log::Level::Info};
toy::log::Backend::instance().push(&meta, 10, 20);

Performance Characteristics

  • push(): O(n) where n is the formatted message length; bounded by LOG_MESSAGE_CAPACITY.
  • setSink() / setTimestampPolicy(): O(1) pointer store.
  • instance(): O(1) static load.
  • Memory usage: one sink pointer plus one function pointer.

Safety Guarantees

  • Singleton invariant: debug assertion fires on a second instance; undefined behavior in release builds if violated.
  • Lifetime contract: the installed sink and any state captured by the timestamp policy must outlive the backend.
  • Exception safety: all operations are noexcept.
  • Thread safety: not thread-safe in v1; serialize push() externally until the ring-buffer transport lands.
Note
Move and copy operations are deleted; instances are pinned to their declaration scope.
Warning
The default timestamp policy returns 0. Install a real policy via setTimestampPolicy() before relying on toy::log::Record::timestamp.

void toy::log::Backend::push ( const Metadata * meta,
const Args &... args )
inlinenoexcept

Formats args into a toy::log::Record and dispatches it to the installed sink.

Stamps the record with timestampPolicy()(), then forwards the call-site format pattern through toy::vformatTo() into the record's message buffer. Synchronously invokes toy::log::ISink::write() on the installed sink. Returns early if no sink is installed.
