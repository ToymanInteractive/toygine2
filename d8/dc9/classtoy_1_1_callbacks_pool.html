Detailed Description

template<typename type, size_t allocatedSize = 4>
class toy::CallbacksPool< type, allocatedSize >

Template callback pool class with fixed-size callback storage.

CallbacksPool is a lightweight, high-performance callback pool class that uses a fixed-size array to store function pointers. It provides a simple observer pattern implementation without dynamic memory allocation, making it suitable for embedded systems, real-time applications, and performance-critical code where memory allocation overhead must be minimized.

Template Parameters
typeThe type of the callback parameter. Must be a copyable type.
allocatedSizeThe maximum number of callbacks that can be stored in the pool. Must be greater than zero (default: 4).

Usage Example

#include "core/callbacks_pool.hpp"
// Define callback function
void onEvent(int value) {
// Handle event
}
// Create a callback pool with default size (4 callbacks)
toy::CallbacksPool<int> pool;
// Subscribe to events
pool.add(onEvent);
// Invoke all registered callbacks
pool.call(42);
// Check subscriber count
auto count = pool.subscribersAmount();
// Unsubscribe
pool.remove(onEvent);
// Clear all subscriptions
pool.reset();

Performance Characteristics

  • ‚öôÔ∏è Construction: O(1) constant time
  • üìù Adding Callback: O(n) where n is the number of registered callbacks (for duplicate check)
  • üîó Removing Callback: O(n) where n is the number of registered callbacks
  • üîç Calling Callbacks: O(n) where n is the number of registered callbacks
  • üìä Subscriber Count: O(1) constant time (cached count)
  • üíæ Memory Usage: Fixed at compile time
  • ‚ö° Cache Performance: Excellent due to stack allocation and contiguous memory layout
  • üìã Copy Performance: Fast due to array-based storage

Safety Guarantees

  • üõ°Ô∏è Contracts & Debug Checks: Null pointer checks are asserted in debug; in release, violating preconditions is UB
  • üîí Bounds Safety: Pool capacity is fixed at compile time; adding beyond capacity is asserted in debug
  • üìê Type Safety: Template parameters ensure type-safe callback invocation
  • ‚ö†Ô∏è Exception Safety: All operations are noexcept, no exceptions thrown
Note
The pool does not prevent duplicate subscriptions - adding the same callback multiple times will only register it once and return true.
When the pool is full, attempts to add new callbacks will fail (asserted in debug mode).
Warning
Callbacks must remain valid for the lifetime of the CallbacksPool object. The pool does not manage callback lifetime.

Adds a callback to the pool.

This method registers a callback function in the pool. If the callback is already registered, the operation succeeds but does not create a duplicate entry. If the pool is full, the operation fails.

Parameters

Returns

Note

If the pool is full, the method returns false and asserts in debug mode.
