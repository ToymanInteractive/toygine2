Detailed Description

template<OStringStreamBackend BackendType>
class toy::FormatContext< BackendType >

Zero-overhead format context that binds a formatting pass to a concrete output backend.

Holds a reference to an toy::OStringStreamBackend storage and forwards write() / put() calls directly to the backend's append / push_back methods.

Template Parameters
BackendTypeThe concrete output storage type. Must satisfy toy::OStringStreamBackend.

Key Features

  • Zero overhead: Calls backend methods directly; no function-pointer indirection.
  • Typed backend access: backend() exposes the underlying storage for inspection or advanced use.
  • constexpr: All operations are constexpr when BackendType allows it.
  • noexcept: Every member is noexcept, matching the toy::OStringStreamBackend contract.

Usage Example

#include "core.hpp"
toy::FixedString<64> buf;
toy::FormatContext<toy::FixedString<64>> ctx(buf);
ctx.write("hello", 5);
ctx.put(' ');
ctx.write("world", 5);
// buf == "hello world"

Performance Characteristics

  • Construction: O(1) constant time; stores one reference.
  • write(): O(n) where n is count; forwards to backend append(const char *, size_t).
  • put(): O(1) constant time; forwards to backend push_back(char).
  • backend(): O(1) constant time; returns a reference.

Safety Guarantees

  • Exception safety: All operations are noexcept (guaranteed by toy::OStringStreamBackend).
  • Lifetime: The BackendType reference must remain valid for the lifetime of the context.
  • Bounds: Overflow behaviour follows BackendType capacity semantics.

Compatibility

  • C++ standard: C++20 or later (concepts, constexpr as implemented).
  • Embedded: Suitable when BackendType uses fixed storage (e.g. toy::FixedString).
