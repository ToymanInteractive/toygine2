105/*!
106 \brief Concept defining the requirements for string-like types.
107
108 This concept defines the interface that any type must satisfy to be considered string-like. It requires the type to
109 have a size() method that returns a value convertible to std::size_t, and a c_str() method that returns a value
110 convertible to const char*.
111
112 (Template parameter T is the type to check for string-like properties.)
113
114 \note Types satisfying this concept can be used with FixString template methods that accept generic string types, such
115 as constructors, assignment operators, and various string manipulation methods.
116
117 \note The concept ensures type safety and provides a compile-time guarantee that the required methods exist and return
118 compatible types.
119*/
120template <typename T>
121concept StringLike = requires(const T & s) {
122 { s.size() } -> std::convertible_to<std::size_t>;
123 { s.c_str() } -> std::convertible_to<const char *>;
124};
