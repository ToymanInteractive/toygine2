Detailed Description

Elapsed-time counter anchored to the active toy::chrono::SteadyClock.

Captures the tick count at construction and computes elapsed time on every call to elapsed() or elapsedTicks() by reading the current tick count from toy::chrono::SteadyClock. Trivially copyable: copying a Stopwatch produces an independent counter with the same start point.

Key Features

  • Trivially copyable: safe to copy with memcpy; no destructor or heap involvement.
  • 8-byte footprint: single int64_t field; suitable for stack, struct-of-arrays, and embedded targets.
  • Zero-overhead elapsed: elapsed() subtracts two tick values with no dynamic allocation or system call overhead beyond the counter read in toy::chrono::SteadyClock::nowTicks().
  • Resettable: reset() moves the anchor to the current tick without constructing a new object.

Usage Example

#include "core.hpp"
toy::chrono::ClockSource clock;
toy::chrono::Stopwatch sw;
// ... work ...
const auto ms = toy::chrono::duration_cast<std::chrono::milliseconds>(sw.elapsed());

Performance Characteristics

  • Construction: O(1); one counter read via toy::chrono::SteadyClock::nowTicks().
  • elapsed(): O(1); one counter read and one subtraction.
  • elapsedTicks(): O(1); one counter read and one subtraction.
  • reset(): O(1); one counter read.
  • Memory usage: 8 bytes.

Safety Guarantees

  • Precondition: an active toy::chrono::ClockSource must exist for the duration of any call; behaviour is undefined if no source is active.
  • Non-negative elapsed: elapsed() and elapsedTicks() return non-negative values as long as the underlying counter is monotonically non-decreasing, which is guaranteed by toy::chrono::SteadyClock.
  • Exception safety: all operations are noexcept.
