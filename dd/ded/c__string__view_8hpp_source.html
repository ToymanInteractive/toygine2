30/*!
31 \class CStringView
32 \brief Non-owning string view class for C-style strings.
33
34 CStringView is a lightweight, non-owning wrapper around C-style strings that provides a safe and efficient way to work
35 with null-terminated character arrays. It offers a std::string_view-like interface while being specifically designed
36 for constexpr contexts and compile-time string manipulation.
37
38 Unlike std::string_view, CStringView is optimized for constexpr usage and provides additional functionality for
39 compile-time string operations. It does not own the underlying string data and simply holds a pointer to it, making
40 it extremely lightweight and efficient.
41
42 \section features Key Features
43
44 - ‚öôÔ∏è **Zero Allocation**: No dynamic memory allocation, only holds a pointer
45 - üîß **constexpr Support**: Most operations can be evaluated at compile time
46 - üõ°Ô∏è **Exception Safety**: All operations are noexcept
47 - üîó **STL Compatibility**: Provides std::string_view/std::string like interface
48 - üåç **UTF-8 Support**: Built-in UTF-8 character counting and manipulation
49 - üß¨ **Type Safety**: Uses C++20 concepts for type safety
50 - üìè **Non-owning**: Does not manage memory, safe for temporary strings
51
52 \section usage Usage Example
53
54 \code
55 #include "c_string_view.hpp"
56
57 // Create a string view from a C string
58 toy::CStringView str("Hello, World!");
59
60 // Use in constexpr context
61 constexpr auto greeting = toy::CStringView("Hello, World!");
62 constexpr auto world = toy::CStringView("World");
63 constexpr auto position = greeting.find(world);
64
65 // Safe string operations
66 if (str.starts_with("Hello")) {
67 // Process the string
68 }
69 \endcode
70
71 \section performance Performance Characteristics
72
73 - ‚öôÔ∏è **Construction**: O(1) - just stores a pointer
74 - üìù **Assignment**: O(1) - just copies a pointer
75 - üìè **Size/Length**: O(n) - requires traversing the string to find the null terminator
76 - üîç **Search Operations**: O(n*m) where n is the string length and m is the pattern length
77 - üíæ **Memory Usage**: Minimal - only stores a single pointer
78 - ‚ö° **Cache Performance**: Excellent due to pointer-only storage
79 - üìã **Copy Performance**: Extremely fast - just pointer copy
80 - üéØ **String Operations**: Optimized for common cases
81
82 \section safety Safety Guarantees
83
84 - üõ°Ô∏è **Contracts & Debug Checks**: Bounds are asserted in debug; in production, violating preconditions is UB
85 - üîí **Null Pointer Safety**: All C-string operations validate pointers via assertions in debug
86 - üìê **Type Safety**: Concepts prevent invalid usage
87 - ‚ö†Ô∏è **Exception Safety**: All operations are noexcept, no exceptions thrown
88 - ‚è∞ **Lifetime Safety**: User must ensure underlying string remains valid
89
90 \section compatibility Compatibility
91
92 - üÜï **C++23**: Requires C++23 or later for full functionality
93 - üîó **STL Integration**: Compatible with STL algorithms and containers
94 - üåê **Cross-Platform**: Works on all platforms supported by the compiler
95 - üîß **Embedded Systems**: Suitable for resource-constrained environments

97 \warning The underlying C string must remain valid for the lifetime of the CStringView object.
