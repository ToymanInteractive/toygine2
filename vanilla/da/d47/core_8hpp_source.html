108/*!
109 \brief Concept defining the requirements for string-like types.
110
111 This concept defines the interface that any type must satisfy to be considered string-like. It requires the type to
112 have a size() method that returns a value convertible to std::size_t, and a c_str() method that returns a value
113 convertible to const char*.
114
115 (Template parameter T is the type to check for string-like properties.)
116
117 \note Types satisfying this concept can be used with FixedString template methods that accept generic string types,
118 such as constructors, assignment operators, and various string manipulation methods.
119
120 \note The concept ensures type safety and provides a compile-time guarantee that the required methods exist and return
121 compatible types.
122*/
123template <typename T>
124concept StringLike = requires(const T & str) {
125 { str.size() } -> std::same_as<std::size_t>;
126 { str.c_str() } -> std::convertible_to<const char *>;
127};
