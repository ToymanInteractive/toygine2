112/*!
113 \brief Concept defining the requirements for string-like types.
114
115 This concept defines the interface that any type must satisfy to be considered string-like. It requires the type to
116 have a size() method that returns a value convertible to std::size_t, and a c_str() method that returns a value
117 convertible to const char*.
118
119 (Template parameter T is the type to check for string-like properties.)
120
121 \note Types satisfying this concept can be used with FixedString template methods that accept generic string types,
122 such as constructors, assignment operators, and various string manipulation methods.
123
124 \note The concept ensures type safety and provides a compile-time guarantee that the required methods exist and return
125 compatible types.
126*/
127template <typename T>
128concept StringLike = requires(const T & str) {
129 { str.size() } -> std::same_as<std::size_t>;
130 { str.c_str() } -> std::convertible_to<const char *>;
131};

198/*!
199 \brief Compile-time assertion function for constexpr contexts.
200
201 This function provides assertion capabilities in constexpr contexts where regular assertions cannot be used. When the
202 \a condition is false, it throws a std::invalid_argument exception with the provided \a message.
203
204 \param condition The condition to check. Must be true for the assertion to pass.
205 \param message The error message to include in the exception if the assertion fails.
206
207 \throws std::invalid_argument If \a condition is false, with \a message as the error description.
208
209 \note This function is designed for use in constexpr contexts where traditional assertions are not available. It
210 should be used sparingly and only when compile-time validation is essential.
211
212 \note Unlike regular assertions, this function always throws an exception in both debug and release builds when the
213 condition fails.
214
215 \see assert
216 \see assert_message
217*/
218constexpr void constexpr_assert(bool condition, const char * message) {
219 if (!condition)
220 throw std::invalid_argument(message);
